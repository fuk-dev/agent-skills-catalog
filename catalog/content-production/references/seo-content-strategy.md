# SEO記事量産戦略（ロングテール設計）

> プロセス4で使用。サイト本体の原稿とは別に、検索流入を獲得するための SEO 記事を計画的に量産する。

---

## 1. SEO記事量産の目的

### 目的

1. **ロングテールでの検索流入獲得**: ビッグキーワードで勝てなくても、ロングテールで勝つ
2. **LLMO（AI検索）対策**: AIに引用される情報源になる
3. **E-E-A-T の強化**: 専門性・権威性・信頼性を示す
4. **リピート訪問の促進**: 役立つ情報があるサイトとして認知
5. **CV 導線**: ブログ → サービスページへの誘導

### SEO 記事量産の効果（目安）

| 施策 | 月間検索流入の増加目安 | 到達時期 |
|---|---|---|
| 月 1 本ペース（年 12 本） | 500-2,000 | 6ヶ月〜1年 |
| 月 2 本ペース（年 24 本） | 1,000-4,000 | 6ヶ月〜1年 |
| 月 4 本ペース（年 48 本） | 2,000-8,000 | 6ヶ月〜1年 |

※ 地域×業種×サービスの3軸で設計した場合

## 2. キーワード戦略

### 3層のキーワード階層

```
【最上層】ビッグキーワード（競争激烈）
  例: 「外壁塗装」「リフォーム」「美容皮膚科」
  → 検索ボリュームは大きいが、上位表示は難しい
  
【中層】ミドルキーワード（地域×業種）
  例: 「那珂川市 外壁塗装」「博多区 美容皮膚科」
  → 中程度のボリューム、地域密着で狙える
  
【最下層】ロングテールキーワード（具体的ニーズ）
  例: 「那珂川市 外壁塗装 補助金」「博多区 シミ取り レーザー 費用」
  → ボリューム小だが CVR 高い、量産で合算効果大
```

### 記事量産で狙うのはロングテール

**ビッグキーワードは TOP・サービスページで**、**ロングテールは記事で**狙う。

### キーワード調査の手法

1. **Google サジェスト**: 検索窓に入力したときの候補
2. **関連検索**: 検索結果下部の「他の人はこちらも検索」
3. **Google キーワードプランナー**: 月間検索ボリューム確認
4. **ラッコキーワード**: 関連キーワードを一括取得
5. **People Also Ask**: 検索結果の質問枠
6. **競合の記事分析**: 上位記事のキーワード構成

### ロングテールキーワードの設計軸

**4軸の掛け合わせ**で膨大なロングテールを生成：

| 軸 | リフォーム業の例 | クリニックの例 |
|---|---|---|
| 地域 | 那珂川市・南区・春日市・大野城市 | 博多区・中央区・早良区 |
| サービス | 外壁塗装・屋根塗装・防水工事 | シミ取り・ホワイトニング・脱毛 |
| 属性・状況 | マンション・戸建・築10年・築30年 | 男性・女性・20代・40代 |
| ニーズ | 費用・相場・補助金・口コミ・期間 | 費用・痛み・期間・副作用 |

**生成パターン**:
- リフォーム: 4地域 × 3サービス × 4属性 × 5ニーズ = **240パターン**
- クリニック: 3地域 × 3施術 × 4属性 × 4ニーズ = **144パターン**

## 3. 記事の型（4種類）

SEO 記事は以下の 4 型で構成。詳細テンプレは `templates/seo-article-templates.md`。

### 型1: ハウツー型（How to）

**検索意図**: 「やり方を知りたい」

**キーワード例**: 「外壁塗装 DIY 手順」「確定申告 個人事業主 やり方」

**構成**:
```
h1: ○○のやり方（完全ガイド）
  リード: 誰向けの記事か、何が分かるか

h2: 準備するもの・前提知識

h2: ステップ1
h2: ステップ2
h2: ステップ3
h2: ステップ4

h2: よくある失敗・注意点

h2: まとめ
```

### 型2: 比較型

**検索意図**: 「A と B、どちらがいいか」

**キーワード例**: 「シリコン塗料 フッ素塗料 比較」「税理士 公認会計士 違い」

**構成**:
```
h1: A と B の違い・比較（2026年最新版）
  リード: どんな人向け、結論の予告

h2: A と B の基本的な違い（表形式）

h2: A の特徴・メリット・デメリット

h2: B の特徴・メリット・デメリット

h2: どちらを選ぶべきか（ケース別）

h2: まとめ・おすすめ
```

### 型3: 解説型

**検索意図**: 「○○とは何か、意味を知りたい」

**キーワード例**: 「インプラント 仕組み」「遺留分 とは」

**構成**:
```
h1: ○○とは？基本から分かりやすく解説

h2: ○○の定義・基本

h2: ○○の種類・分類

h2: ○○のメリット・デメリット

h2: ○○を検討するときのポイント

h2: ○○に関するよくある質問

h2: まとめ
```

### 型4: 事例型

**検索意図**: 「実際の事例・ケーススタディを見たい」

**キーワード例**: 「○○市 外壁塗装 事例」「中小企業 税務調査 対応事例」

**構成**:
```
h1: 事例: ○○市A様邸の外壁塗装（築25年）

h2: お客様情報（表形式）

h2: ご依頼のきっかけ

h2: 現地調査・診断結果

h2: 施工内容・工程

h2: 施工前後の比較

h2: 費用・期間

h2: お客様の声

h2: 担当者コメント

CTA: 類似案件のお問い合わせ
```

## 4. 年間記事量産計画の立て方

> **実運用シート**: カテゴリ別・ファネル別・事業別の配分設計と月次消化トラッキングは `templates/seo-annual-balance-sheet.md` を使用すること。

### Step 1: キーワードリスト作成

4軸（地域×サービス×属性×ニーズ）の掛け合わせで、**200-500 パターン**のキーワードリストを作成。

### Step 2: 優先度付け

全キーワードを以下で評価：

| 項目 | 評価 |
|---|---|
| 月間検索ボリューム | 10以下 / 10-100 / 100-1,000 / 1,000以上 |
| 競合の強さ | 弱・中・強 |
| CV につながる可能性 | 高・中・低 |
| 記事執筆の難易度 | 低・中・高 |

**優先度マトリクス**:
- 検索ボリューム中・競合弱・CV 高 → 最優先
- 検索ボリューム大・競合強・CV 高 → 2番目
- 検索ボリューム小・競合弱・CV 低 → 量産対象

### Step 3: 執筆スケジュール

優先度順に年間計画：

| 6月 | 4本 | 那珂川市・春日市エリア記事 |
| 7月 | 4本 | 補助金情報・地域別特集 |
| 8月 | 4本 | 夏の屋根塗装記事 |
| 9月 | 4本 | 台風対策・外壁点検 |
| 10月 | 4本 | 秋の外装メンテナンス |
| 11月 | 4本 | 年末前の塗装検討記事 |

## 5. 記事執筆の鉄則（SEO 特化）

### タイトル（title タグ）の最適化

- **主要キーワードを含む**
- **32 字以内**（SERP で切れない）
- **年号・数字を入れる**（例: 「2026年最新版」「5つのポイント」）
- **感情に訴える言葉**（例: 「完全ガイド」「徹底比較」「失敗しない」）

### 例

| 改善前 | 改善後 |
|---|---|
| 外壁塗装の基本 | 外壁塗装の基本知識【2026年版】費用・期間・業者選び |
| シリコン塗料について | シリコン塗料 vs フッ素塗料 徹底比較！失敗しない選び方 |
| インプラントとは | インプラントとは？メリット・デメリット・費用を完全解説 |

### メタディスクリプションの最適化

- **90-120字**
- **主要キーワードを1-2回含める**
- **結論 or 要約を先に**
- **CTA を末尾に**

### 例

```
外壁塗装の基礎知識を徹底解説。費用相場・塗料の種類・業者選びのコツ・
補助金まで、初めての方に必要な情報を網羅。2026年最新の費用相場も掲載。
まずは無料見積もりから。
```

### 本文の最適化

1. **冒頭 100 字以内で結論**（AI検索で引用されやすい）
2. **見出しに主要キーワードを自然に配置**
3. **箇条書き・表を活用**（読みやすさ＋SEO）
4. **画像には alt 属性**（キーワードを含める）
5. **内部リンクを 3-5 箇所**（関連ページへ誘導）
6. **外部リンクは信頼できるソースのみ**（政府・専門機関等）

### FAQ セクション（実在する疑問がある場合のみ）

FAQ は固定レシピで機械的に付けない。**実在する読者の疑問**（受付・現場で実際に聞かれる質問、GSC クエリ由来）があり、**独自の具体的な回答**を提供できる場合にのみ設ける。

- 問数は固定しない（実在する疑問の数だけ。埋めるための水増し Q&A を作らない）。
- 露出・引用・回遊を目的化しない（`non-commodity-content-standard.md` §3「やらなくていいこと」参照）。

### 構造化データの対応

実装前に必ず Google Search Gallery（https://developers.google.com/search/docs/appearance/structured-data/search-gallery）で現行サポートと利用条件を確認する（**schema.org 上の妥当性と Google の検索機能サポートは別物**）。

| 記事タイプ | 対応構造化データ |
|---|---|
| 全記事 | Article + BreadcrumbList |
| ハウツー型 | Article のみ（HowTo リッチリザルトは 2023 年に廃止済み） |
| 比較型 | Article のみ |
| 事例型 | Article（自社・自組織を対象とし自社が管理するレビューは Google の star review feature の対象外＝self-serving。医療・美容医療・歯科では体験談規制が優先＝`industry-presets.md` クリニック節 FB-02/FB-03 参照） |

> **FAQPage について**: FAQ リッチリザルトは 2026-05-07 に Google 検索での表示終了（Search Console レポート・リッチリザルトテスト対応は 2026-06、Search Console API 対応は 2026-08 に段階削除）。**AI 検索効果を目的とした新規追加はしない**。既存 markup の一斉削除は不要——ページ改修時に可視 FAQ との一致と利用目的を確認して整理し、内容不一致・誤りはその時点で修正・削除する。

## 6. 記事量産のワークフロー

### 1記事の標準ワークフロー

```
【Day 1】 キーワード調査・ブリーフ作成（1-2 時間）
  ↓
【Day 2】 構成設計・リサーチ（2-3 時間）
  ↓
【Day 3-4】 本文執筆（4-6 時間）
  ↓
【Day 5】 校正・編集（1-2 時間）
  ↓
【Day 6】 画像手配・挿入位置指示（1 時間）
  ↓
【Day 7】 納品・実装依頼
```

**1記事あたり 10-15 時間程度**（量産体制構築後）。

### 量産体制の組み方

#### パターン A: 社内完結

- **ディレクター**: キーワード選定、ブリーフ作成
- **ライター**（1名）: 執筆 4-6 本／月
- **編集者**: 校正・SEO チェック
- **デザイナー**: 画像手配

#### パターン B: 外部ライター活用

- **ディレクター**: キーワード選定、ブリーフ作成
- **外部ライター**（2-5名）: 各 2-4 本／月
- **編集者**（社内）: ブリーフ精度チェック、納品物校正
- **デザイナー**: 画像手配

#### パターン C: AI + ライター

- **AI ツール**: ドラフト生成
- **ライター**: AI 出力を推敲・加筆
- **編集者**: 事実確認・校正
- **注意**: AI 丸投げは品質・SEO ペナルティリスク

公開3ヶ月後に以下を確認：

```
[ ] 目標順位を達成したか
[ ] 検索流入があったか
[ ] 次のアクションへの遷移があったか
[ ] 記事内で離脱が多い箇所がないか
[ ] 改善余地があるか
```

順位が伸びない記事は、**リライト**を検討。

## 8. リライトの判断基準

既存記事の見直しタイミング：

| 状況 | 対応 |
|---|---|
| 公開3ヶ月、圏外 | 即時リライト（キーワード再選定） |
| 公開6ヶ月、50位以下 | リライト（内容強化） |
| 公開1年、20位以下 | リライト（最新情報反映） |
| 定期点検 | 半年〜年1回 |

### リライトの手法

1. **競合記事の再分析**: 現時点で上位の記事を確認
2. **キーワード再選定**: より具体的なロングテールに変更
3. **情報の最新化**: 2026年時点の最新情報を反映
4. **見出し構造の見直し**: 情報の網羅性向上
5. **画像・動画の追加**: コンテンツを豊富化
6. **FAQ の追加**: 検索意図に応える情報を補強

## 9. よくある失敗と対策

### 失敗1: 検索意図と記事が合っていない

**症状**: 「外壁塗装 相場」で検索する人に、施工事例を見せている

**対策**:
- 記事執筆前にキーワードで実際に検索
- 上位記事の構成・内容を確認
- 検索意図（情報収集・比較検討・購入意欲）を判定

### 失敗2: キーワード詰め込み

**症状**: 不自然に「外壁塗装」を30回繰り返し

**対策**:
- 自然な文章で必要に応じてキーワードを配置
- 類義語・関連語を使う（「塗装工事」「塗り替え」等）
- Google のアルゴリズムは文脈を理解する

### 失敗3: E-E-A-T 不足

**症状**: 医療・金融記事なのに、誰が書いたか分からない

**対策**:
- 執筆者・監修者プロフィールを明示
- 資格・経歴・実績を記載
- 根拠となる出典を明記

### 失敗4: 量産に走って品質低下

**症状**: 月10本以上の無理な量産で、どれも薄っぺらい

**対策**:
- 無理ない本数に調整（月2-4本が現実的）
- 量より質を優先
- ライターを増やすより、1記事に時間をかける
