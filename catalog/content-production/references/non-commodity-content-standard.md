# Non-Commodity Content Standard（非コモディティ・コンテンツ標準）

> content-production の中心思想。初期制作・公開後運用の**両方**で本標準に従う。
> 一言: **主体は人間、AI は構成・整理・表現の補助。価値の源泉は一次情報・現場知見・実体験・独自データ・専門家視点。**

---

## 1. 基本方針

- **AI だけで一般論を量産しない。**
- 主体は人間、AI は構成・整理・表現の補助とする。
- コンテンツの価値は、**一次情報・現場知見・実体験・独自データ・専門家視点**に置く。
- AI 生成文は**人間が確認**してから公開する。
- ハルシネーション・未確認情報・誇張表現を避ける。
- 過度な AI 検索向けの文章分割や会話型フレーズ変換は不要。
- テクニカル SEO は**土台**であり、上位表示や成果は**コンテンツの質・関連性・事業貢献**で評価する。
- アクセス数だけでなく、**問い合わせ・予約・成約・来院などの事業 KPI** で評価する。

## 2. Google が問題視するのは「AI 利用」そのものではない

問題視されるのは以下:
- AI が主体で 0 から生成しただけのコンテンツ
- 人間のチェックなしにそのまま投稿するコンテンツ
- 大量生成コンテンツのばらまき
- ハルシネーションの未確認
- 実在しない実績・数値・口コミ・事例の創作

→ したがって content-production の原則:
- AI だけで一般論を量産しない
- 一次情報なしで断定しない
- 実績・数値・効果・比較は**根拠確認**する（`claim-register` に記録。数値・調査系は必須8項目＝時期/母数/方法/定義/除外条件/実施主体/根拠資料/更新日を記録し、調査条件を本文にも併記する）
- 人間レビューを前提にする
- ハルシネーションチェックを必須にする（`content-qa-checklist`）
- 一般論だけのページは**再取材・再設計へ戻す**

## 3. やらなくていいこと（過剰最適化の回避）

- AI のためだけの過度な段落分割
- 無理な会話型フレーズへの書き換え
- 100点満点の HTML 文法への過剰なこだわり
- 第三者 SEO ツール指標の盲信
- ドメインオーソリティ等を Google 公式指標のように扱うこと
- 実在する疑問・独自回答のない定型 Q&A の量産（固定問数で機械的に FAQ を付けない。`seo-content-strategy.md`「FAQ セクション」参照）

重要なのは、**ユーザーの疑問に自然言語で正確に答える内容がコンテンツ内にあること**。

## 4. テクニカル SEO の位置づけ（土台）

1. **クロール** — Googlebot がページを発見・収集
2. **インデックス** — 収集したページを登録・理解
3. **サービング** — クエリに最適なページを配信

ただし **クロール・インデックス ≠ 上位表示**。上位表示・成果を決める中心は、コンテンツの質・関連性・独自性・事業貢献。

## 5. 非コモディティ化に必要な情報（一次情報の引き出し）

- 独自アンケート / 調査データ
- 顧客の声
- 営業・現場の知見
- よくある質問（受付・現場で実際に聞かれる）
- 検証・実体験レポート
- 専門家ならではの分析
- 失敗例 / 改善例
- 現場での判断基準
- 成約理由 / 失注理由

→ これらを **`content-brief`** で制作前に集め、**`claim-register`** で根拠を記録し、**`content-qa-checklist`** で公開前に検証する。

## 6. 評価 KPI（運用時）

アクセス数だけで評価しない。見るべき指標:
問い合わせ数 / 予約数 / 来院・来店数 / 成約数 / CVR / CTA クリック率 / 電話タップ / LINE クリック / フォーム到達率 / フォーム完了率 / GSC 表示回数・クリック数・CTR・平均順位 / 重要ページの流入 / 事業貢献度。

> 検索流入が増えても、問い合わせ・予約・成約につながらなければ「改善成功」とは言わない。
