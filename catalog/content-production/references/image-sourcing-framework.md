# 画像調達判断フレームワーク（4手段）

> プロセス5で使用。各画像をどう調達するかを、目的・コスト・品質の観点で判断する。

---

## 1. 4つの調達手段

```
【手段1】既存素材
  └─ クライアントが保有している写真・イラスト
  
【手段2】買取素材（ストックフォト）
  └─ iStock / Adobe Stock / Shutterstock 等
  
【手段3】新規撮影
  └─ プロカメラマン or 自社スタッフで撮影
  
【手段4】AI 画像生成
  └─ Midjourney / DALL-E / Stable Diffusion 等
```

---

## 2. 判断フロー

画像の用途に応じて以下のフローで判断：

```
[画像の用途確認]
  ↓
[実物・実在が必要？]
  ↓ Yes → 施工事例・スタッフ・店舗・症例・物件
  │       ↓
  │       [既存素材あり？]
  │         ↓ Yes → 【手段1: 既存素材】
  │         ↓ No  → 【手段3: 新規撮影】
  │
  ↓ No → 抽象的イメージ・背景・アイコン
        ↓
        [オリジナリティ必要？]
          ↓ Yes → 【手段4: AI生成】 or 【手段3: 新規撮影】
          ↓ No  → 【手段2: 買取素材】
```

## 4. 手段1: 既存素材

### メリット

- コスト 0
- クライアント実物を表現できる
- 即座に使用可能

### デメリット

- 品質がまちまち
- 解像度不足の可能性
- 著作権・肖像権の確認必須
- 古い情報の可能性（店舗リニューアル済み等）

### チェック項目

```
[ ] 解像度は十分か（最低 1920px 幅）
[ ] 著作権は誰が持っているか（クライアント or 撮影者）
[ ] 肖像権はクリアか（人物が写っている場合）
[ ] 撮影時期は古くないか
[ ] 構図・明るさは合格レベルか
[ ] 現状（店舗・商品）と一致しているか
```

### 使い方

1. クライアントから素材提供を受ける（DropBox / Google Drive 等）
2. 使える素材・使えない素材を選別
3. 必要に応じて画像補正（明るさ・色調）
4. 新規撮影が必要なカットをリストアップ

## 5. 手段2: 買取素材（ストックフォト）

### 主要サービス

| サービス | 特徴 | 価格目安 |
|---|---|---|
| **iStock / Getty Images** | 高品質、企業向け | 1枚 3,000-10,000円 |
| **Adobe Stock** | Adobe 製品連携 | 月額プラン 3,000円〜 |
| **Shutterstock** | 豊富な品数 | 月額プラン 3,000円〜 |
| **Unsplash** | 無料、高品質 | 無料（商用可） |
| **Pexels** | 無料、商用可 | 無料 |
| **Pixta** | 日本人モデル豊富 | 1枚 3,000-5,000円 |
| **写真AC** | 無料（要登録） | 無料（枚数制限あり） |

### 使い分け

| 予算 | 推奨 |
|---|---|
| 無料 | Unsplash / Pexels / 写真AC |
| 月数千円 | Adobe Stock / Shutterstock（サブスク） |
| 都度購入 | iStock / Pixta |

### メリット

- 手軽、即日使える
- 高品質
- 法的リスクが低い（規約遵守）

### デメリット

- 他社と被る可能性
- 実物・実在ではない
- 日本人モデル不足（海外系サービス）
- 「ストックフォト臭」が出る

### 使用時の注意点

1. **ライセンス確認**:
   - 商用利用可か
   - 改変可か
   - クレジット表記必須か
   - 再配布・再販売禁止か
2. **モデルリリース**: 人物が写っている場合、モデルリリース済みか
3. **プロパティリリース**: 特定建物・商品が写っている場合、プロパティリリース済みか

### 避けるべきストックフォトの特徴

- **明らかにストック臭**: 白背景でポーズを取っている日本人グループ
- **他社多用**: 同じ素材が競合サイトでも使われている
- **時代遅れ**: 画質・ファッションが古い
- **業種と合わない**: 歯科医院に洋風レストラン風の写真

## 6. 手段3: 新規撮影

### 撮影の種類

| 種類 | 用途 | 予算目安 |
|---|---|---|
| プロカメラマン | 高品質な訴求素材 | 半日 5-15万、1日 10-30万 |
| カメラマンアシスタント付き | ライティング重視 | 半日 10-25万、1日 20-50万 |
| 自社撮影 | 日常的な更新素材 | 機材のみ |
| スマホ撮影 | SNS・日常更新 | 実質無料 |

### 新規撮影が必要な場面

- **施工事例**: ビフォー・アフター、工程写真
- **スタッフ紹介**: ポートレート
- **店舗・施設**: 外観・内観・設備
- **商品・料理**: 物撮り
- **インタビュー**: スタッフ・お客様の声

### 撮影指示書の必須項目

詳細は `templates/photo-shoot-direction.md` を参照：

- 撮影目的・用途
- 撮影リスト（何をどれだけ撮る）
- 構図ガイド
- ライティング指示
- 被写体ディレクション（人物の場合）
- 撮影場所・時間帯
- 納品形式

### カメラマン選定のポイント

- **業種への理解**: 建築撮影経験、人物撮影経験等
- **作風**: ナチュラル、シャープ、温かい等、ブランドと合うか
- **ポートフォリオ**: 過去作品の質
- **価格**: 予算内に収まるか
- **納品スピード**: 撮影後の編集・納品までの期間

## 7. 手段4: AI 画像生成

### 主要サービス（2026年時点）

| サービス | 特徴 | 価格目安 |
|---|---|---|
| **Midjourney** | 高品質、芸術的 | 月額 $10-60 |
| **DALL-E 3** | ChatGPT 連携 | ChatGPT Plus 月額 $20 |
| **Stable Diffusion** | ローカル実行可、自由度高 | 無料〜月額 |
| **Adobe Firefly** | 商用利用安全、Adobe 連携 | 月額プラン |
| **Leonardo AI** | ゲーム系・イラスト系 | 無料〜月額 |

### AI 生成が向いている用途

- **抽象的なイメージ画像**（ヒーロー画像等）
- **背景・パターン**
- **アイコン・イラスト**
- **架空のコンセプト表現**
- **OGP 画像のベース**

### AI 生成が不適切な場面（絶対NG）

- **実物・実在の表現**:
  - 施工事例
  - スタッフ写真
  - 店舗写真
  - 商品写真（実物）
  - 症例（医療）
  - 物件（不動産）
- **特定人物の表現**（肖像権侵害リスク）
- **ブランドロゴ・既存キャラクター**（著作権侵害リスク）

### 商用利用ガイド

#### 7-A. サービス別ライセンス詳細比較（2026年時点）

| サービス | 商用利用 | 著作権帰属 | 補償制度 | クレジット表記 | 特記事項 |
|---|---|---|---|---|---|
| **Midjourney** | 有料プランで可 | ユーザーに帰属 | なし | 不要 | 無料プランは商用不可。年商100万ドル以上の企業はProプラン以上必須 |
| **DALL-E 3** | 可 | ユーザーに所有権 | なし | 不要 | OpenAIの利用規約に準拠。API経由も同様 |
| **Stable Diffusion** | 可（モデル依存） | モデルライセンス次第 | なし | モデル依存 | SDXLはApache 2.0で商用可。サードパーティモデルは個別確認必須 |
| **Adobe Firefly** | 可（商用安全） | ユーザーに帰属 | **あり**（IP補償） | 不要 | Adobe Stock・パブリックドメインのみで学習。最も安全 |
| **Leonardo AI** | 有料プランで可 | ユーザーに帰属 | なし | 不要 | 無料プランは商用利用不可 |
| **Canva AI** | 可（Proプラン） | Canva規約に準拠 | なし | 不要 | Canva Pro内での生成は商用可。無料プランは制限あり |
| **Ideogram** | 有料プランで可 | ユーザーに帰属 | なし | 不要 | テキスト描画が得意。ロゴ・バナー用途に適す |

#### 7-B. 商用利用判断フロー

```
[AI画像を商用利用したい]
  ↓
[有料プランで生成したか？]
  ↓ No → ❌ 商用利用不可（ほとんどのサービス）
  ↓ Yes
[実在の人物・ブランド・ロゴを含むか？]
  ↓ Yes → ❌ 使用不可（肖像権・商標権侵害リスク）
  ↓ No
[学習データの出典は明確か？]
  ↓ No → ⚠️ リスクあり（Adobe Firefly 等の安全なサービスを推奨）
  ↓ Yes
[クライアントにAI生成であることを説明済みか？]
  ↓ No → ⚠️ 事前説明を推奨（特にYMYL業種）
  ↓ Yes
✅ 商用利用可能（実務チェックリストを実行）
```

#### 7-C. 日本法でのAI生成画像の著作権整理

| 論点 | 現状の整理 |
|---|---|
| **AI生成物の著作物性** | 日本の著作権法では「思想又は感情を創作的に表現したもの」が著作物。AIが自律的に生成したものは著作物と認められない可能性が高い |
| **プロンプトの創作性** | プロンプトに創作的寄与（試行錯誤・細かい指示）があれば、生成物に著作物性が認められる余地がある（文化庁見解） |
| **第三者の著作権侵害** | AIが学習データに含まれる既存著作物に類似した画像を生成した場合、依拠性が認められれば著作権侵害となる可能性 |
| **実務上の推奨** | 「著作権で保護されない可能性がある」ことを前提に、サービス利用規約での権利保護を優先する |

> **参考**: 文化庁「AIと著作権に関する考え方について」（2024年）、知的財産戦略本部「AI時代の知的財産権検討会」議事録

#### 7-D. AI生成画像の開示義務に関する動向

| 地域・規制 | 概要 | HP制作への影響 |
|---|---|---|
| **EU AI Act** | 高リスクAIシステムに透明性義務。ディープフェイク禁止 | 現時点ではHP用のイメージ画像に直接の開示義務なし。今後の拡大に注意 |
| **日本** | AI生成物の開示義務は現時点でなし | 任意だが、YMYL業種（医療・士業・不動産）では信頼性の観点から開示を推奨 |
| **GoogleのE-E-A-T** | AI生成コンテンツ自体は禁止していないが、「人間が確認・編集」を推奨 | 実物が必要な場面でAI画像を使うとE-E-A-T評価が下がるリスク |

#### 7-E. 実務チェックリスト（生成→確認→記録→使用の4ステップ）

**Step 1: 生成**
```
[ ] 有料プランで生成したか
[ ] プロンプトに実在の人物・ブランド・キャラクターを含めていないか
[ ] 差別的・不快な表現を含んでいないか
```

**Step 2: 確認**
```
[ ] 不自然な歪みがないか（手の指・顔のパーツ・文字等）
[ ] 解像度は十分か（最低 1920px 幅）
[ ] 業種に合ったトーンか
[ ] ブランドカラーに調和するか
[ ] 既存の著作物に類似していないか（Google画像検索で確認）
```

**Step 3: 記録**
```
[ ] 使用サービス名・プランを記録
[ ] プロンプト全文を保存（再現性確保）
[ ] 生成日時を記録
[ ] 画像メタデータに「AI生成」を記載（推奨）
[ ] 画像管理表（image-asset-list.md）に「AI生成」と明記
```

**Step 4: 使用**
```
[ ] クライアントにAI生成であることを説明済みか
[ ] 使用用途は「AI生成が向いている用途」に該当するか
[ ] 「AI生成が不適切な場面」に該当していないか
[ ] YMYL業種の場合、信頼性への影響を検討したか
```

> **判例・規制は急速に変化中。年に1回以上、最新情報を確認し本セクションを更新すること。**

### AI 画像の品質チェック

```
[ ] 解像度は十分か（最低 1920px 幅、SD 等の高解像度機能使用）
[ ] 不自然な歪みがないか（手の指・顔のパーツ等）
[ ] 業種に合ったトーンか
[ ] ブランドカラーに調和するか
[ ] テキスト部分（看板・書籍等）が意味不明になっていないか
[ ] 差別的・不快な表現がないか
[ ] 商用利用可のプランで生成したか
```

## 8. 画像品質の基準（全手段共通）

### 解像度

- **最低**: 1920x1080 px（FullHD）
- **推奨**: 2560x1440 px 以上
- **印刷用途**: 300dpi 以上

### ファイル形式

| 用途 | 推奨形式 | 代替 |
|---|---|---|
| Web 掲載（一般） | WebP | JPEG |
| ロゴ・アイコン | SVG | PNG（透過） |
| 写真（一般） | WebP | JPEG |
| イラスト | SVG / WebP | PNG |
| アニメーション | WebP（アニメ） | GIF / MP4 |

### 圧縮

- **JPEG**: 品質 80-90%
- **WebP**: 品質 85%
- **PNG**: 透過必要時のみ（JPEG より重い）

### ファイル名

統一した命名規則：

```
[section]-[purpose]-[number].[ext]

例:
- top-hero-01.webp
- service-gaiheki-01.webp
- works-case001-before-01.webp
- works-case001-after-01.webp
- staff-ceo-01.webp
- og-image-default.jpg
```

## 9. 画像のブランド統一

サイト全体で画像のトーンを揃えることが重要：

### 統一すべき要素

- **明るさ**: 全体的に明るめ / 暗め
- **彩度**: 高め / 低め
- **コントラスト**: 強め / 弱め
- **色調**: 暖色系 / 寒色系
- **構図**: 中心配置 / 三分割法

### 統一方法

1. **撮影時に統一指示**（撮影指示書に明記）
2. **同じ色調補正をかける**（Photoshop のアクション等）
3. **買取素材も事後編集**（色調・明るさ調整）
4. **AI 生成は同じプロンプトスタイル**

## 10. 画像の権利管理

### 確認必須項目

| 項目 | 確認内容 |
|---|---|
| 著作権 | 誰が持っているか、使用許諾はあるか |
| 肖像権 | 人物が写っている場合の本人許諾 |
| プロパティライセンス | 特定建物・商品等 |
| 商標権 | ロゴ・商品名の使用 |

### 権利関係の書類

新規撮影時は以下を取得：

- **撮影契約書**（カメラマンとの契約）
- **モデルリリース**（人物が写る場合、本人の使用許諾）
- **プロパティリリース**（特定建物・施設の使用許諾）
- **著作権譲渡合意**（必要な場合）

### よくあるトラブル

1. **スタッフ退職後の写真継続使用**: 退職時に使用可否を契約明記
2. **施工事例の施主許諾**: 施工契約時に「事例として公開可」の承諾を得る
3. **ストックフォトの再配布**: 禁止されているので再配布しない
4. **AI 生成画像の著作権**: サービスによって異なる、最新情報確認

## 11. コスト感の目安

### プロジェクト全体の画像予算目安

| 案件規模 | 画像予算の目安 | 内訳例 |
|---|---|---|
| 小規模（〜50万） | 0-5万円 | ストックフォト＋既存素材 |
| 中規模（50-300万） | 10-40万円 | 撮影半日＋ストックフォト |
| 大規模（300万〜） | 30-100万円 | 撮影複数日＋プロモーション動画 |

### 節約テクニック

- **既存素材の活用**: クライアント保有素材を最大限活用
- **撮影を一度にまとめる**: 複数ページ分を同日に撮影
- **無料ストックフォト**: Unsplash / 写真AC
- **AI 生成の併用**: 抽象的な画像のみ AI で節約
- **買取素材のサブスク**: 年間プランで単価を下げる
