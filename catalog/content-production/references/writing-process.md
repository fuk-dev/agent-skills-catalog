# 原稿執筆プロセス詳細

> プロセス3で使用。企画→執筆→校正の全工程を体系化し、品質と量産性を両立する。

---

## 1. 執筆プロセスの全体像

```
【ステップ1】 前提整理
  └─ ページの目的・ターゲット・CV・キーワードを確認

【ステップ2】 構成設計
  └─ 見出し階層（h1〜h3）の設計、本文の流れ

【ステップ3】 本文執筆
  └─ テンプレートに沿って執筆

【ステップ4】 SEO・LLMO 最適化
  └─ キーワード配置、構造化データ対応、結論ファースト

【ステップ5】 CTA 設計
  └─ 次のアクションへの誘導配置

【ステップ6】 校正
  └─ 誤字脱字、文体統一、法令違反、根拠チェック

【ステップ7】 納品
  └─ Markdown形式、Front Matter付き
```

---

## 2. ステップ1: 前提整理

### 着手前に確認する項目

原稿執筆の前に、以下を確認する。ユーザーの依頼内容から抽出し、不足があればユーザーに質問する:

| 項目 | 内容 |
|---|---|
| ページの目的 | 訴求 / 理解 / 信頼形成 / CV / SEO / 法令遵守 |
| ターゲット | メインペルソナの属性・課題 |
| 最優先 CV | 問合せ/予約/購入/資料請求 |
| 主要キーワード | h1・導入・見出しに配置 |
| 文字数目安 | 最低○○字〜 |
| ブランドトーン | 親しみ/高級/誠実/革新的 |
| NGワード | 使わない表現 |
| 業種の法令 | 業種プリセット (`references/industry-presets.md`) から遵守必須の規制 |

### 前提が不明な場合の対応

- **ターゲット不明**: ユーザーに再確認、ペルソナを明確化
- **CV 不明**: ユーザーに確認、仮設定は禁止
- **キーワード不明**: SEO目標キーワードが設定されていない → 設定してから着手
- **文字数目安不明**: 推奨文字数で作成（TOP 1,500-2,000字、下層 1,000-3,000字、事例 1,500-2,000字）

## 3. ステップ2: 構成設計

### 基本の構成要素

原稿には必ず以下を含める：

1. **h1**（ページタイトル）: 主要キーワード含む、32字以内
2. **リード文**: 150-200字で誰向け・何を解決するかを明示
3. **本文**（h2・h3階層）: 論理的な順序、キーワードを自然に配置
4. **CTA**: 次のアクション誘導
5. **関連情報**（必要に応じて）: 関連ページ・補足情報

### 見出し階層の設計原則

- **h1 は 1 ページに 1 つ**（複数配置しない）
- **h1 → h2 → h3 の順**（h3 から始めない、h2 を飛ばさない）
- **見出しだけで全体像が伝わる**（見出しを拾い読みして意味が通る）
- **主要キーワードを h1 と h2 の少なくとも 1 つに含む**

### ページタイプ別の構成パターン

#### TOP ページ

```
h1: ファーストビューキャッチコピー
  リード文（課題共感＋解決策）

h2: 私たちの強み（3-5 ポイント）
  h3: 強み1
  h3: 強み2
  h3: 強み3

h2: サービス紹介（主力3-4サービス）
  h3: サービスA
  h3: サービスB
  h3: サービスC

h2: 事例（3-5件ピックアップ）

h2: よくある質問（3-5問）

CTA: お問い合わせ / 資料請求
```

#### サービス詳細ページ

```
h1: サービス名 + 対応地域

h2: このサービスの概要（100-150字）

h2: 選ばれる理由・特徴（3-5点）
  h3: 理由1
  h3: 理由2
  h3: 理由3

h2: サービスの流れ
  h3: ステップ1
  h3: ステップ2
  h3: ステップ3

h2: 料金・プラン
  h3: プランA
  h3: プランB

h2: 事例

h2: よくある質問

CTA: お問い合わせ
```

#### 事例ページ

```
h1: 事例タイトル（地域×サービス×特徴）

h2: お客様情報（表形式）

h2: ご依頼のきっかけ・課題

h2: 実施した施工内容・サービス内容
  h3: 工程1
  h3: 工程2
  h3: 工程3

h2: 施工前後の比較 / 成果

h2: お客様の声

h2: 担当者紹介

CTA: 類似案件のお問い合わせ
```

#### FAQページ

```
h1: よくあるご質問

h2: カテゴリ1（サービスについて）
  ### Q1. 質問文
    A. 回答文
  ### Q2. ...

h2: カテゴリ2（料金について）
  ### Q6. 質問文
    A. 回答文

h2: カテゴリ3（期間について）

CTA: その他の質問はお問い合わせ
```

## 4. ステップ3: 本文執筆

### 執筆の原則

1. **一文は 60-80 文字を目安**（長すぎる文は読まれない）
2. **主語を省略しすぎない**（何の話か分かる明示）
3. **接続詞を適切に使う**（「しかし」「たとえば」「このため」等）
4. **難しい単語を避ける**（中学生でも理解できる語彙）
5. **結論ファースト**（結論→根拠→詳細の順）

### 定型文・NGワード排除

以下の表現を**絶対に使わない**：

| NG表現 | 理由 | 代替案 |
|---|---|---|
| 「誠実・丁寧に」 | 他社も使う定型文 | 具体的な行動で表現「見積書には項目ごとに理由を記載」 |
| 「お客様の立場に立って」 | 固有情報ゼロ | 「過去3年で平均○件の相談に対応」 |
| 「高品質」 | 根拠不明 | 「5年保証付き」「○○認定取得」 |
| 「安心・安全」 | 中身がない | 「事前カウンセリング60分無料」「万一の際の○○保証」 |
| 「業界ナンバーワン」 | 景表法リスク | 「○○調査で満足度○%（2024年）」 |
| 「何でも対応」 | 専門性が見えない | 特定分野を明示「相続に特化」 |
| 「最適な」「最善の」 | 主観的 | 客観的な指標で示す |

### 数字で語る

抽象表現を具体的な数字に置き換える：

| 抽象 | 具体 |
|---|---|
| 豊富な実績 | 累計3,500件の施工実績 |
| 長年の経験 | 創業20年、業界歴15年 |
| 多くのお客様 | 年間1,200件のご依頼 |
| 業界トップクラス | 2023年の○○調査で対応満足度90% |
| 短期間で対応 | 最短48時間で初回対応 |
| 広いエリア対応 | 福岡県内7市2町で対応 |
| 手頃な価格 | 10万円〜50万円のレンジ |

### 固有名詞を入れる

- **地名**: 「福岡市南区那珂川」「博多駅徒歩3分」
- **製品名・メーカー名**: 「日本ペイント パーフェクトトップ」
- **資格・認定**: 「一級建築士」「税理士登録番号第○○号」
- **連携サービス**: 「Google Ads パートナー認定」

固有名詞は**情報量**を上げ、**検索エンジンからの評価**も上げる。

### 読み手の動詞で書く

「クライアントは何ができるか」を動詞で：

- 「無料で見積もりを比較できます」
- 「24時間いつでも予約できます」
- 「LINE で相談できます」
- 「現地を確認したうえで判断できます」

## 5. ステップ4: SEO・LLMO 最適化

### SEO 最適化

1. **キーワード配置**:
   - h1 に主要キーワード（地域×サービス）
   - h2 に関連キーワード
   - 導入文に主要キーワード
   - 本文内に自然に 5-10 回
   - **キーワード詰め込みは厳禁**（ペナルティリスク）

2. **内部リンク**:
   - 関連ページへのリンクを文中に配置
   - アンカーテキストに意味のある文字列（「こちら」「詳細」ではなく「外壁塗装の施工事例」等）

3. **メタ情報**:
   - title（30-32字）: 主要キーワード含む
   - meta description（90-120字）: 数字・固有名詞・CTA

### LLMO（AI検索最適化）

AI（ChatGPT・Perplexity・Google AI Overview）に引用されやすい書き方：

1. **結論ファースト**: 最初の 200 字で結論を書く
2. **数値データを入れる**: AI は数値を引用しやすい
3. **FAQ形式**: 質問→回答のペア
4. **固有名詞**: 会社名・地域名を明記
5. **著者情報**: 監修者・執筆者の経歴・資格

### 構造化データを意識した書き方

以下のコンテンツは、後から構造化データ（JSON-LD）を付けやすい形で書く：

| コンテンツ | 構造化データ | 書き方のポイント |
|---|---|---|
| FAQ | （FAQPage は新規非推奨） | Q・A は可視 FAQ として書く。FAQPage リッチリザルトは 2026-05-07 表示終了のため AI 検索目的の新規 markup は付けない（詳細は `seo-content-strategy.md` §5） |
| 事例 | Article | タイトル・公開日・更新日・著者・要約 |
| サービス | Service | 名称・説明・提供エリア・価格 |
| 口コミ | （Review は self-serving 対象外） | 自社が管理するレビューは Google の star review feature の対象外。医療・美容医療・歯科は体験談規制が優先（`industry-presets.md` FB-02/FB-03） |
| 会社情報 | LocalBusiness / Organization | NAP情報を揃える |

## 6. ステップ5: CTA 設計

### CTA の基本

- **すべてのページに CTA を配置**
- **CTA は具体的な行動を示す**（「お問い合わせ」より「無料見積もりを依頼」）
- **ページ末尾＋中盤にも配置**（長いページでは複数箇所）

### CTA のパターン

| ページタイプ | 推奨 CTA |
|---|---|
| TOP | 主要CV（問合せ/予約）＋サブCV（事例を見る/資料請求） |
| サービス詳細 | そのサービスへの問合せ |
| 事例 | 類似案件の問合せ |
| FAQ | 解決しない場合はお問い合わせ |
| ブログ記事 | 関連サービスページへの誘導 |
| 会社概要 | 採用情報 or お問い合わせ |

### CTA コピーの例

#### 問合せ系

- 「無料で見積もりを依頼する」
- 「30秒で入力完了。まずは相談する」
- 「実物を見てから判断したい方へ。現地調査を依頼」

#### 資料請求系

- 「施工事例集（PDF）を無料でダウンロード」
- 「料金プラン一覧を請求する」

#### 予約系

- 「オンライン予約（24時間受付）」
- 「初診予約・カウンセリング予約はこちら」

## 7. ステップ6: 校正

### 校正チェックリスト

```
【誤字脱字】
[ ] 誤字・脱字がないか
[ ] 固有名詞のスペル・漢字が正しいか
[ ] 数字・日付が正確か
[ ] 電話番号・URL が正しいか

【文体統一】
[ ] 「です・ます」「だ・である」が統一されているか
[ ] 漢字・ひらがな表記の統一（「問い合わせ」or「問合せ」等）
[ ] 半角・全角スペースの統一
[ ] カギカッコ・ダブルクォートの統一

【論理性】
[ ] 見出しと本文が一致しているか
[ ] 段落ごとにテーマが明確か
[ ] 結論→根拠→詳細の順になっているか
[ ] 矛盾する記述がないか

【根拠】
[ ] 数値に出典があるか
[ ] 「ナンバーワン」「最高」等に根拠があるか
[ ] 事実と意見が区別されているか

【法令遵守】
[ ] 業種プリセットの NG表現を使っていないか
[ ] 業法違反がないか
[ ] 薬機法・景表法違反がないか
[ ] 医療広告等ガイドラインに違反していないか
[ ] 著作権違反（無断引用）がないか

【SEO・LLMO】
[ ] 主要キーワードが h1 と導入に入っているか
[ ] メタ情報が揃っているか
[ ] 構造化データに対応した書き方か
[ ] 内部リンクが適切か

【CTA】
[ ] 次のアクションが明示されているか
[ ] CTA が目立つ位置・文言か
```

### 校正のコツ

- **一晩寝かせる**: 書いた直後は見落としが多い
- **声に出して読む**: リズムの悪い文は違和感で分かる
- **他人の目を入れる**: 書き手以外の目線で校正
- **逆から読む**: 誤字脱字発見に効果的

## 8. ステップ7: 納品

### 納品形式（Markdown + Front Matter）

```markdown
---
title: "ページタイトル（title タグ用、30-32字）"
description: "ページのメタディスクリプション（90-120字）"
h1: "ページの h1（32字以内）"
url: /path/to/page/
keywords: ["主要キーワード", "関連キーワード1", "関連キーワード2"]
author: "執筆者名（内部名でも可）"
publishDate: "YYYY-MM-DD"
updateDate: "YYYY-MM-DD"
category: "ページカテゴリ"
targetWordCount: 2000
actualWordCount: 1980
---

# h1: ページタイトル

## リード文
（150-200字）

## h2: 見出し1

### h3: 小見出し1

本文...

### h3: 小見出し2

本文...

## h2: 見出し2

...

## CTA
（次のアクションへの誘導）
```

### 納品時の添付情報

原稿本体とともに以下を明示：

1. **使用画像の指示**（どの位置にどの画像を配置するか）
2. **内部リンク一覧**（リンク先URL・アンカーテキスト）
3. **構造化データの想定**（FAQ・Article・Service 等）
4. **校正済みか**（チェックリスト完了か）
5. **法令遵守確認済みか**

## 9. 文字数の目安

| ページタイプ | 最低 | 標準 | 上限 |
|---|---|---|---|
| TOP | 1,200字 | 1,500-2,000字 | 3,000字 |
| 会社概要 | 500字 | 800-1,200字 | 2,000字 |
| サービス詳細 | 1,500字 | 2,000-3,000字 | 5,000字 |
| 事例ページ | 1,000字 | 1,500-2,000字 | 3,000字 |
| FAQ | 1,500字（Q&A合算） | 3,000-5,000字 | 10,000字 |
| ブログ記事 | 1,500字 | 2,500-4,000字 | 8,000字 |
| アクセス | 300字 | 400-600字 | 1,000字 |
| プライバシーポリシー | 定型 | 1,000-2,000字 | 3,000字 |

**文字数は品質ではない**。必要な情報が十分伝わるかが重要。

## 10. よくある失敗と対策

### 失敗1: 情報を詰め込みすぎ

**症状**: 全ページ 3,000字以上、読みにくい

**対策**:
- 各ページ 1 テーマに絞る
- 詳細情報は別ページに分ける
- 見出しで整理して拾い読みできるように

### 失敗2: 文体が途中で変わる

**症状**: 「です・ます」と「だ・である」が混在

**対策**:
- 書き始める前に文体を決める
- 校正時に統一チェック

### 失敗3: ターゲットがブレる

**症状**: 一般消費者向けの文と、BtoB 向けの文が同じページに混在

**対策**:
- ページごとにターゲット（メインペルソナ）を明確化
- ターゲット不明の場合は書かない（先にペルソナ確定）

### 失敗4: CTA が消える

**症状**: 記事を読み終わっても、次に何をすればいいか分からない

**対策**:
- 必ず末尾に CTA
- 長い記事は中盤にも CTA
- CTA コピーを具体化（「お問い合わせ」より「無料見積もり依頼」）

### 失敗5: 法令違反

**症状**: 医療サイトで「絶対治る」、不動産で免許番号なし

**対策**:
- 業種プリセットを必ず確認
- 校正時に法令チェック項目を入れる
- 弁護士・監修者チェック（医療系・金融系）
